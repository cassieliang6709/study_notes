# 附录：面试开口模板与口语回答稿

## 通用开口模板

题面很业务化时，可以先这样说：

> 我先忽略业务术语，把问题抽象一下。这里本质上是一个……

然后接：

- 有向图判环 / 拓扑排序问题
- 二维 grid 上的无权最短路问题
- 多路归并问题
- 表达式解析或嵌套上下文处理问题
- 连续子数组的滑动窗口问题
- 区间重叠和资源分配问题

## 为什么这个算法对

### 拓扑排序

“如果图里有环，就不可能完成所有任务。Kahn 算法每次处理一个入度为 0 的节点，相当于执行一个当前没有前置依赖的任务。最后如果能处理完所有节点，就说明图无环。”

### Grid BFS

“因为每次移动一步的代价都一样，所以这是无权图最短路。BFS 按层扩展，第一次到终点时一定是最短距离。”

### Heap

“我只需要维护当前候选中的最值，不需要保存所有元素的全局顺序，所以堆最合适。”

### Stack

“括号或嵌套结构意味着有多层上下文。栈最适合保存进入一层前的状态，并在退出时恢复。”

### Sliding Window

“这是连续区间优化问题。窗口只会向右移动，所以每个元素最多被左右指针访问一次，能做到线性时间。”

## 复杂度怎么讲

### 拓扑排序

- 时间：`O(V + E)`
- 空间：`O(V + E)`

### Grid BFS / DFS

- 时间：`O(m * n)`
- 空间：`O(m * n)`

### Heap

- 多路归并：`O(N log k)`
- Top K：`O(n log k)`

### Stack

- 时间：`O(n)`
- 空间：`O(n)`

### Sliding Window

- 时间：`O(n)`
- 空间：`O(k)` 或字符集大小

## follow-up 怎么自然接

### Topo

- 如果只判断能不能完成：看处理节点数是否等于总节点数
- 如果要字典序最小：队列换最小堆
- 如果问 DFS 能不能返回顺序：后序加入 + reverse

### Grid

- 允许斜走：改 `dirs`
- 多个起点：multi-source BFS
- 能打破墙：状态 BFS
- 权重不同：Dijkstra / 0-1 BFS

### Stack

- 加 `* /`：当前层栈即时处理乘除
- 多位数：持续构造 `num`
- 日志 start/end：调用栈

### Intervals

- 端点相等算不算冲突：先问清题意
- 最大并发时刻：扫描线
- 为什么用堆：堆顶是最早结束区间
