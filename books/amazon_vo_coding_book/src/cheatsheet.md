# 附录：一页速记 / 3 天冲刺顺序

## 一页速记

- 依赖关系、能否完成、输出顺序：`拓扑排序`
- grid 最短路：`BFS`
- grid 连通块：`DFS/BFS flood fill`
- 多个有序流、Top K：`heap`
- 表达式、括号、嵌套：`stack`
- 查存在、做映射、频率统计：`set/map`
- 连续子串子数组：`sliding window`
- 一圈一圈打印矩阵：`四边界模拟`
- 区间重叠、会议室：`排序 + heap / merge`
- 链表原地操作：`dummy + 三指针 + 快慢指针`
- 最近不超过 / 第一个不小于：`ordered map / binary search`

## 最后建议

如果时间真的不多，优先把下面这些讲熟：

- 为什么 `Course Schedule` 用拓扑排序
- 为什么 grid 最短路用 `BFS`
- 为什么 `Merge K Sorted Lists` 用堆
- 为什么 `Basic Calculator` 用栈
- 为什么连续子串题常用滑窗
- 为什么最长连续序列用 `set`
