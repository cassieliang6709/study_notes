# Project Guide: MiniMind

原项目：

- https://github.com/jingyaogong/minimind

---

## 这个项目最适合你学什么

`MiniMind` 适合当“从零训练小语言模型”的第一站。

它的价值不在于追求 SOTA，而在于：

- 让你在个人资源可承受范围内理解 LLM 主流程
- 让你第一次把“模型、训练、推理”看成一个整体

---

## 为什么应该在这个阶段做它

如果你已经做了一些应用层项目，但对模型内部仍然模糊，这个项目很合适。

它比课程作业更容易下手，但又比纯概念讲解更具体。

---

## 前置知识

- Python
- PyTorch 基础
- 张量、梯度、优化器基本概念
- transformer 有最基础认知更好

---

## 先看什么

建议不要先试图读完整仓库。

推荐顺序：

1. README 的项目定位
2. tokenizer 相关部分
3. 模型定义
4. training script
5. inference / chat / generation 相关脚本

---

## 第一周最小任务

第一周只要做到：

1. 跑通环境
2. 看懂项目里“最小模型”是怎么组织的
3. 跑一次最简单训练或推理
4. 画出你自己的模块图

这就够了。

---

## 开做指南

### 环境要求

- 一台能跑 PyTorch 的机器
- 最好有 GPU，但第一遍重点是理解流程

### 最小启动动作

1. 克隆仓库
2. 安装依赖
3. 找到最小训练或推理入口
4. 跑一个最小示例

### 第一遍不要改什么

- 不要先调大模型尺寸
- 不要先追求训练效果
- 不要先换复杂数据集

### 第二遍建议你改什么

- 改 tokenizer 或 vocab 流程
- 改模型宽度 / 层数
- 跑一个更小的自定义语料实验

---

## 你做完后应该掌握什么

- 知道一个小语言模型训练的基本部件
- 知道从文本到 token 再到 logits 的链路
- 知道训练脚本和推理脚本分别做什么

---

## 常见卡点

- 把“会运行”误以为“看懂了”
- 只盯 attention，忽略 tokenizer 和 data pipeline
- 一开始就想训很久、训很大

---

## 最小完成标准

- 你能说清楚这个仓库的训练入口在哪里
- 你能解释 tokenizer、model、loss、generation 的关系
- 你能改一个小超参并观察变化
