## 1. Trapping Rain Water

**题型识别**

```text
双指针 / Two Pointers
```

**中文解释**

这题可以用单调栈，也可以用双指针。  
双指针更直观：

- 左边能装多少水取决于 `left_max`
- 右边能装多少水取决于 `right_max`

每次先处理较低的一边，因为它的装水上限已经确定。

**English Explanation**

This problem can be solved with either a monotonic stack or two pointers.  
The two-pointer solution is more intuitive:

- water on the left depends on `left_max`
- water on the right depends on `right_max`

We always process the lower side first because its water limit is already determined.

**Python 标准解法**

```python
from typing import List


class Solution:
    def trap(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        left_max = right_max = 0
        ans = 0

        while left < right:
            if height[left] < height[right]:
                left_max = max(left_max, height[left])
                ans += left_max - height[left]
                left += 1
            else:
                right_max = max(right_max, height[right])
                ans += right_max - height[right]
                right -= 1

        return ans
```

## 2. Longest Valid Parentheses

**题型识别**

```text
栈 + 下标 / Stack with Indices
```

**中文解释**

栈里不直接存括号，而是存下标。  
初始化放一个 `-1`，表示“上一个无法匹配的位置”。  
这样每当找到一个合法区间时，就可以用：

```text
当前下标 - 栈顶下标
```

计算区间长度。

**English Explanation**

The stack stores indices, not the actual parentheses.  
Initialize it with `-1`, which represents the position before the start of the current valid substring.  
Then whenever we match a valid pair, the length can be computed by:

```text
current_index - stack_top
```

**Python 标准解法**

```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        stack = [-1]
        ans = 0

        for i, ch in enumerate(s):
            if ch == "(":
                stack.append(i)
            else:
                stack.pop()
                if not stack:
                    stack.append(i)
                else:
                    ans = max(ans, i - stack[-1])

        return ans
```

## 3. Valid Parentheses

**题型识别**

```text
括号匹配 / Parenthesis Matching
```

**中文解释**

这是最基础的栈题。  
遇到左括号就入栈，遇到右括号就检查栈顶是否是对应的左括号。

**English Explanation**

This is the basic stack matching problem.  
Push opening brackets onto the stack, and when a closing bracket appears, check whether the stack top matches it.

**Python 标准解法**

```python
class Solution:
    def isValid(self, s: str) -> bool:
        mp = {')': '(', ']': '[', '}': '{'}
        stack = []

        for ch in s:
            if ch in mp:
                if not stack or stack.pop() != mp[ch]:
                    return False
            else:
                stack.append(ch)

        return not stack
```

## 4. Simplify Path

**题型识别**

```text
路径栈模拟 / Stack-based Path Simulation
```

**中文解释**

Unix 路径里：

- `.` 表示当前目录，忽略
- `..` 表示回到上一级，弹栈
- 普通字符串表示进入子目录，入栈

最后把栈里的目录重新拼起来即可。

**English Explanation**

In Unix paths:

- `.` means current directory, so ignore it
- `..` means go to parent, so pop from the stack
- a normal token means enter that directory, so push it

At the end, join the stack back into a canonical path.

**Python 标准解法**

```python
class Solution:
    def simplifyPath(self, path: str) -> str:
        stack = []

        for part in path.split("/"):
            if part == "" or part == ".":
                continue
            if part == "..":
                if stack:
                    stack.pop()
            else:
                stack.append(part)

        return "/" + "/".join(stack)
```

## 5. Basic Calculator

**题型识别**

```text
表达式栈 / Expression Stack
```

**中文解释**

这题只有：

- `+`
- `-`
- 括号

核心是维护当前表达式的：

- `result`
- `sign`

遇到 `(` 时把当前上下文压栈，遇到 `)` 时再恢复。

**English Explanation**

This version only contains:

- `+`
- `-`
- parentheses

The main idea is to maintain the current:

- `result`
- `sign`

When seeing `(`, push the current context onto the stack; when seeing `)`, restore it.

**Python 标准解法**

```python
class Solution:
    def calculate(self, s: str) -> int:
        stack = []
        result = 0
        num = 0
        sign = 1

        for ch in s:
            if ch.isdigit():
                num = num * 10 + int(ch)
            elif ch in "+-":
                result += sign * num
                num = 0
                sign = 1 if ch == "+" else -1
            elif ch == "(":
                stack.append(result)
                stack.append(sign)
                result = 0
                sign = 1
            elif ch == ")":
                result += sign * num
                num = 0
                result *= stack.pop()
                result += stack.pop()

        return result + sign * num
```

## 6. Basic Calculator II

**题型识别**

```text
即时结算 / Immediate Evaluation
```

**中文解释**

这题没有括号，但多了乘除。  
技巧是：

- `+` / `-` 的结果先压栈
- `*` / `/` 立刻和栈顶结合结算

最后把栈里所有数相加。

**English Explanation**

This version has no parentheses, but includes multiplication and division.  
The trick is:

- for `+` / `-`, push numbers onto the stack
- for `*` / `/`, immediately combine with the stack top

At the end, sum up the stack.

**Python 标准解法**

```python
class Solution:
    def calculate(self, s: str) -> int:
        stack = []
        num = 0
        op = "+"

        for i, ch in enumerate(s):
            if ch.isdigit():
                num = num * 10 + int(ch)

            if ch in "+-*/" or i == len(s) - 1:
                if op == "+":
                    stack.append(num)
                elif op == "-":
                    stack.append(-num)
                elif op == "*":
                    stack.append(stack.pop() * num)
                else:
                    prev = stack.pop()
                    stack.append(int(prev / num))

                op = ch
                num = 0

        return sum(stack)
```

## 7. Basic Calculator III

**题型识别**

```text
递归表达式解析 / Recursive Expression Parsing
```

**中文解释**

这是前两题的综合版，既有括号，也有乘除。  
常见写法是递归：

- `helper()` 负责处理当前括号层级
- 遇到 `(` 时递归进入子表达式
- 遇到 `)` 时返回当前层结果

**English Explanation**

This is the combined version of the previous two calculators: it has both parentheses and multiplication/division.  
A common implementation is recursive:

- `helper()` parses the current parenthesis level
- on `(`, recurse into a sub-expression
- on `)`, return the result of the current level

**Python 标准解法**

```python
class Solution:
    def calculate(self, s: str) -> int:
        i = 0

        def helper():
            nonlocal i
            stack = []
            num = 0
            op = "+"

            while i < len(s):
                ch = s[i]

                if ch.isdigit():
                    num = num * 10 + int(ch)

                if ch == "(":
                    i += 1
                    num = helper()

                if ch in "+-*/)" or i == len(s) - 1:
                    if op == "+":
                        stack.append(num)
                    elif op == "-":
                        stack.append(-num)
                    elif op == "*":
                        stack.append(stack.pop() * num)
                    elif op == "/":
                        stack.append(int(stack.pop() / num))

                    num = 0
                    op = ch

                if ch == ")":
                    return sum(stack)

                i += 1

            return sum(stack)

        return helper()
```

## 8. Decode String

**题型识别**

```text
嵌套解码 / Nested Decoding with Stack
```

**中文解释**

当遇到 `[` 时，说明当前字符串和重复次数要暂存起来，进入新层级。  
当遇到 `]` 时，从栈里弹出之前的状态，把当前字符串按次数展开再接回去。

**English Explanation**

When we see `[`, it means the current string and repeat count should be saved before entering a deeper level.  
When we see `]`, we pop the previous context from the stack and expand the current substring accordingly.

**Python 标准解法**

```python
class Solution:
    def decodeString(self, s: str) -> str:
        stack = []
        cur_num = 0
        cur_str = ""

        for ch in s:
            if ch.isdigit():
                cur_num = cur_num * 10 + int(ch)
            elif ch == "[":
                stack.append((cur_str, cur_num))
                cur_str = ""
                cur_num = 0
            elif ch == "]":
                prev_str, num = stack.pop()
                cur_str = prev_str + num * cur_str
            else:
                cur_str += ch

        return cur_str
```

## 9. Design Circular Queue

**题型识别**

```text
循环数组队列 / Circular Array Queue
```

**中文解释**

这题的关键是不要真的移动数组元素。  
只维护：

- `head`
- `count`

尾部位置可以通过：

```text
(head + count) % k
```

算出来。

**English Explanation**

The key is that we never physically shift array elements.  
We only maintain:

- `head`
- `count`

The tail position can be computed by:

```text
(head + count) % k
```

**Python 标准解法**

```python
class MyCircularQueue:
    def __init__(self, k: int):
        self.q = [0] * k
        self.k = k
        self.head = 0
        self.count = 0

    def enQueue(self, value: int) -> bool:
        if self.isFull():
            return False
        tail = (self.head + self.count) % self.k
        self.q[tail] = value
        self.count += 1
        return True

    def deQueue(self) -> bool:
        if self.isEmpty():
            return False
        self.head = (self.head + 1) % self.k
        self.count -= 1
        return True

    def Front(self) -> int:
        return -1 if self.isEmpty() else self.q[self.head]

    def Rear(self) -> int:
        return -1 if self.isEmpty() else self.q[(self.head + self.count - 1) % self.k]

    def isEmpty(self) -> bool:
        return self.count == 0

    def isFull(self) -> bool:
        return self.count == self.k
```

## 10. Exclusive Time of Functions

**题型识别**

```text
调用栈模拟 / Call Stack Simulation
```

**中文解释**

栈顶函数是当前正在运行的函数。  
当新函数开始时，之前的函数先累计运行时间。  
当函数结束时，把自己从栈里弹出，并更新结束时间。

**English Explanation**

The function on top of the stack is the one currently running.  
When a new function starts, the previous top function accumulates time first.  
When a function ends, it is popped from the stack and its finishing interval is recorded.

**Python 标准解法**

```python
from typing import List


class Solution:
    def exclusiveTime(self, n: int, logs: List[str]) -> List[int]:
        ans = [0] * n
        stack = []
        prev = 0

        for log in logs:
            fid, typ, time = log.split(":")
            fid, time = int(fid), int(time)

            if typ == "start":
                if stack:
                    ans[stack[-1]] += time - prev
                stack.append(fid)
                prev = time
            else:
                ans[stack.pop()] += time - prev + 1
                prev = time + 1

        return ans
```
