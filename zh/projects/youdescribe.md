# YouDescribe

生产实习：无障碍工具开发和 AI 辅助音频描述创作。

## 项目概述

YouDescribe 是一个让志愿者为 YouTube 视频撰写音频描述的平台，帮助视障用户理解视觉内容。实习期间，我开发并上线了两个功能：一个暴露了志愿者工作流中隐藏质量问题的播放速度模拟器，以及降低高质量描述撰写门槛的 AI 辅助创作工具。

## 技术栈

- **前端**：React、TypeScript
- **TTS**：Google Cloud Text-to-Speech API
- **后端**：Node.js（现有平台）
- **AI**：LLM 辅助描述草稿生成（集成进现有创作流程）

## 播放速度模拟器

**问题**：志愿者按 1x 速度审听自己的描述，这对大多数视力正常的人来说很自然。但很多真实的视障用户因为长期训练，实际会用 2x 到 3x 的速度去听。一个在 1x 下听起来清晰的描述，在 3x 下可能显得急促、难以跟上。志愿者使用的质量标准，并不是真正用户所需的质量标准。

**解决方案**：用 React + TypeScript 开发了一个播放速度模拟器，接入现有的 Google Cloud TTS 流水线。提交前，志愿者必须以 3x 速度试听自己的描述。产品文案的选择很重要：我把它写成"预览用户实际会怎么听到你的描述"，而不是"质量检查"。这些志愿者在做很善意的事情——这个体验不应该让人感觉被评判。

**结果**：志愿者立刻开始自我修正。他们会自己发现描述在 3x 下太急促，然后主动修改。描述长度下降，质量分上升，用户侧反馈也更好了——无需任何人工审核介入。

## AI 辅助创作

在描述创作流程中集成了 LLM 草稿步骤，降低志愿者从零开始创作的门槛。草稿以"起点"而非"答案"的形式呈现——UI 设计上更鼓励编辑而非直接接受，因为志愿者撰写的描述在细腻度和情感把握上始终优于模型原始输出。

## 关键技术说明

- 播放速度模拟器需要将 TTS 音频流与视频时间轴精确同步，让志愿者能在上下文中评估描述效果，而不是孤立地听。
- 无障碍约束同样适用于工具本身——创作界面需要对同样使用屏幕阅读器的志愿者可用。
- 实习期间上线至生产环境；功能现已在 YouDescribe 平台上线。

## 收获

让创造内容的人亲身体验到用户所经历的一切，是在规模化场景下提升内容质量最可靠的方式之一。志愿者不需要质量系统来告诉他们问题在哪——他们需要的是亲耳听到 3x 速度是什么感觉。听到之后，他们自己就修了。
