# Agent RL 为什么比 SFT 难

我在做 VisoCode 的时候，遇到了一个很典型的 agent 问题：多个 agent 互相调用，没有明确的终止条件，系统在复杂任务上会无限循环。当时的修法是用 LangGraph 加状态管理和条件边，强制控制流程。

这让我对一件事有了直觉：多步 agent 的行为，靠 prompt 控制是有天花板的。在某些场景下，agent 本身需要被训练——不是训练它"怎么说话"，而是训练它"怎么在多步交互中做决策"。

这就是 Agent RL 要解决的问题。

## 先说 SFT 在 agent 场景下的局限

SFT 的训练模式是：给一个输入，学一个输出。

对于单步任务，这很合适：给问题，学答案；给代码描述，学代码。

但多步 agent 任务不是这个结构。

想象一个 agent 要完成"帮我调研这个话题，总结成报告"的任务。它需要：

1. 决定先搜什么
2. 看搜索结果，决定要不要深入某个方向
3. 发现一个新线索，调整搜索策略
4. 汇总所有信息，生成报告

这个过程里，没有"给一个输入，对应一个正确输出"。每一步的决策，依赖前一步的结果；每一步的"对不对"，只有在整个任务完成之后才能评估。

SFT 最多能让 agent 模仿一个"示例轨迹"，但它很难让 agent 学会：当中间步骤的结果和预期不一样时，怎么调整策略。

## Agent RL 的核心结构

Agent RL 的训练不是针对单个（输入，输出）对，而是针对**轨迹**（trajectory）。

一个轨迹是：从初始状态开始，agent 做了一系列动作，最后到达终止状态。整个过程结束后，给这条轨迹一个奖励。

训练目标是：让 agent 学会产生能获得更高奖励的轨迹。

这和 SFT 有几个根本区别：

**奖励是稀疏的。** SFT 每一步都有监督信号；Agent RL 通常只在任务结束时才给奖励。中间每一步对最终结果的贡献，模型需要自己学会估计——这叫 credit assignment。

**状态是动态变化的。** Agent 的每一步动作会改变环境状态，下一步面对的输入也因此变化。模型需要学会根据当前状态做决策，而不是对固定输入产生固定输出。

**探索是必要的。** 要学会好的策略，agent 必须先尝试不同的行动路径。但探索需要成本，探索太少学不到好策略，探索太多效率很低。

## Credit Assignment 是核心难点

这是 Agent RL 里我觉得最难的概念之一。

一个多步任务，假设一共做了 10 步，最后任务失败了。那么这 10 步里，哪几步是关键错误？是第 3 步的搜索策略不对，还是第 7 步的信息汇总出了问题，还是两者都有？

人可以事后分析。但模型需要从奖励信号里自动学会这件事——在轨迹里，哪些步骤对最终结果贡献最大（或最有害）。

这在长轨迹里尤其困难，因为早期的错误可能要到很多步之后才体现出来。

**解法**：通常是引入折扣奖励（对时间更近的步骤给更高权重），或者用 value function 来估计每个状态的长期价值，而不是只看最终奖励。

## 奖励函数怎么设计

另一个难点是：奖励怎么定义？

对于有明确答案的任务——比如数学题（答案对不对）、代码（能不能跑通）——奖励信号相对清晰。

对于开放式任务——比如写一篇报告——"好不好"很难量化。你可以用另一个模型来打分（这相当于重新引入奖励模型的问题），或者用人类来评估（成本高，规模化难）。

奖励函数设计不好的后果是 **reward hacking**：agent 学会了"钻奖励函数的空子"，做出在奖励指标上得高分但实际没有用的行为。

比如一个被训练成"对话轮数越多，奖励越高"的 agent，可能会学会故意把任务拖长，而不是高效完成。

## 多步和单步的不同训练难度

用一张表来对比：

| | SFT | Agent RL |
|---|---|---|
| 训练信号 | 每一步都有（对应正确输出） | 稀疏（通常只有最终奖励）|
| 任务结构 | 单步映射 | 多步序列决策 |
| 中间步骤的正确性 | 直接定义 | 需要 credit assignment |
| 状态变化 | 固定输入 | 动态变化 |
| 探索 | 不需要 | 必须，但有代价 |

## 为什么 Prompt 不能完全代替训练

一个常见的问题是：既然 agent 已经很强了，为什么不能只靠精心设计的 prompt？

答案是：对于很多任务，prompt 已经足够。但对于需要大量试错和策略调整的复杂任务，prompt 能给的只是初始行为偏好，而不能让 agent 从经验中学习和改进。

训练过的 agent 和只靠 prompt 的 agent 的区别，类似于经过训练的运动员和只听过教练讲解的运动员——讲解很重要，但真正的技能需要从实际练习中形成。

## 小结

Agent RL 比 SFT 难，难在这几个地方：

1. **轨迹而非单步**：训练信号来自整条轨迹的结果，而不是每一步的标准答案
2. **Credit assignment**：很难自动判断多步任务里哪一步是关键
3. **探索与利用的平衡**：要学好策略，必须尝试不同路径，但探索有代价
4. **奖励函数设计**：对开放式任务，定义"好不好"本身就是难题

这些难点不是当前技术能完全解决的，它们也是 agent 训练领域还在活跃研究的方向。
