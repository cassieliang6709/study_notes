# Attention 到底在算什么

Attention 是我反复看、反复觉得自己懂了、又反复发现没真懂的东西。

我找到了一个对我来说最有效的理解方式：从一个具体句子出发，一步一步跟着走，不跳到数学符号上。

## 从一个问题开始

考虑这个句子：

> 动物没有过马路，因为它太累了。

"它"指的是什么？是动物，还是马路？

你读完之后可能立刻知道答案是"动物"——因为"太累了"更符合动物的属性。但你大脑做这个判断，用的是什么？

你大脑在做的事情，大致是：

1. 看到"它"
2. 往前扫描，找可能的指代对象
3. 根据"太累了"这个线索，判断"动物"比"马路"更相关
4. 把"动物"的信息纳入对"它"的理解

Attention 机制就是在做这件事——用结构化的方式让每个词"关注"其他词。

## Q、K、V 是什么

Attention 里有三个矩阵：Query（Q）、Key（K）、Value（V）。

直觉上可以这样理解：

- **Query**：我想问什么。"它"在说：我要找我的指代对象。
- **Key**：每个词在说：我是什么类型的信息，你来匹配看看。
- **Value**：如果你选中了我，你能从我这里拿走什么内容。

计算过程大致是：

1. 用"它"的 Query，去和每个词的 Key 做点积
2. 点积越大，说明这两个词越相关
3. 把点积的结果做 softmax，变成概率分布（权重）
4. 用这个权重对所有词的 Value 做加权求和
5. 得到的结果，就是"它"在考虑了所有其他词之后的新表示

所以"它"处理完之后，不再是一个孤立的向量，而是融合了整个句子上下文信息的向量——其中"动物"贡献的权重最高。

## 为什么需要多个 head

单个 attention head 在一次计算里只能关注一种关系。

但语言里的关系是多种多样的：语法关系、语义关系、指代关系、时态关系……

**Multi-head attention** 是让多个 attention head 并行运行，每个 head 学习关注不同类型的关系。比如一个 head 专门学指代关系，一个 head 专门学动词和宾语的关系，最后把所有 head 的结果拼在一起。

这就像你看一篇文章时，同时在追语法、追意思、追逻辑——不是顺序处理，而是并行的。

## 为什么叫 Self-attention

这里的"self"是说：每个 token 的 Query、Key、Value 都来自同一个序列本身。

区别于 cross-attention——那是在两个不同序列之间做 attention，比如 encoder-decoder 结构里，decoder 关注 encoder 的输出。

在纯 decoder 的语言模型里，用的是 self-attention，每个 token 只关注它前面的 token（masked attention），因为训练时不能"看到未来"。

## 为什么 Attention 这么重要

在 Attention 出现之前，处理序列的主流方法是 RNN。RNN 按顺序处理每个词，把信息压缩进一个固定大小的"隐藏状态"里往前传。

这有一个致命问题：如果句子很长，前面的信息传到后面的时候已经被稀释了。

Attention 完全绕开了这个问题：每个 token 都可以直接"看到"序列里任何位置的其他 token，不需要经过中间状态传递。长距离依赖变得一样容易处理，不再随距离衰减。

这就是为什么 Transformer 在处理长文本时比 RNN 效果更好的根本原因。

## 计算代价

Attention 有一个代价：它的计算复杂度是 O(n²)，n 是序列长度。

每个 token 都需要和其他所有 token 做一次点积，序列越长，计算量增长越快。这就是为什么语言模型的上下文长度是有限制的，以及为什么处理长文档是一个持续被研究的工程问题。

最近出现的很多 attention 变体（sparse attention、linear attention、flash attention）都在不同程度上缓解这个问题，但核心计算逻辑和上面说的一样。

## 小结

Attention 的本质是一个**动态的加权聚合机制**：

- 每个 token 生成 Q、K、V
- 用 Q 和其他 token 的 K 计算相关性权重
- 用权重对所有 token 的 V 做加权求和
- 得到一个融合了上下文的新表示

理解了这一点，Transformer 的很多设计选择就变得直觉了：为什么 multi-head、为什么 masked attention、为什么位置编码（因为 attention 本身不感知顺序）。
