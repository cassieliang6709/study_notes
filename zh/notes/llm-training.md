# 语言模型是怎么被训练出来的

我很长一段时间都停在"会调 API"的阶段。我知道怎么用 GPT-4 回答问题，知道怎么写 prompt，但当有人问我"这个模型到底是怎么工作的"，我能给出的答案很模糊。

真正把这件事搞清楚是在学 MiniMind 和 CS336 之后。我想把我理解的过程写下来——不用数学公式，只讲直觉。

## 语言模型在学什么

语言模型预训练的目标，本质上只有一件事：

**给你一段文字，预测下一个词是什么。**

就这么简单。给模型看"今天天气很"，它要学会预测下一个词更可能是"好"还是"差"还是"热"。

这件事看起来简单，但如果你要在海量文本上把它做得很准，模型就必须学会：
- 语法和句法结构
- 词语之间的语义关系
- 常识和世界知识
- 长距离的上下文依赖

所有这些，都是通过预测下一个词这一个任务逼出来的。这就是预训练强大的原因。

## 文字怎么进入模型

模型处理的不是原始文字，而是 token。

**Tokenizer** 负责把文字切成 token。Token 不等于词，一个词可能是一个 token，也可能被切成多个 token，一个 token 也可能是一个字母或一个标点符号。

不同的 tokenizer 切法不同，中文和英文的处理方式也不一样。Tokenizer 是在训练语言模型之前就已经训练好的，它有自己的词表，通常包含几万到十几万个 token。

Token 进入模型之后，第一步是变成向量，这叫做 **embedding**。你可以把 embedding 理解成：把每个 token 映射到一个高维空间里的坐标，语义相近的 token 在这个空间里会离得更近。

## Attention 是核心

模型处理一段文字时，每个 token 不是孤立被处理的，它需要参考其他 token。

**Self-attention** 负责让每个 token "看"其他 token，并根据相关性决定吸收多少信息。

直觉上：当模型处理"它"这个词时，它需要知道"它"指的是什么，所以要往前看，找到最相关的名词。Attention 机制就是在做这件事——动态地决定当前 token 应该关注哪些其他位置。

多个 attention head 并行运行，每个 head 关注不同的语义关系，最后合并。这就是 Transformer 的核心。

## 训练过程是怎么进行的

有了模型结构，训练的过程大致是：

1. 从数据里取一段文字
2. 把前 n 个 token 喂给模型
3. 模型预测第 n+1 个 token 的概率分布
4. 和真实答案比较，算出误差（loss）
5. 通过反向传播，调整模型参数，让下次预测更准
6. 重复这个过程，重复几千亿次

Loss 是衡量"预测有多差"的指标。Loss 下降，意味着模型在预测下一个词这件事上越来越准。

大模型的训练数据量和计算规模是惊人的。GPT-3 用了 3000 亿个 token 训练，参数量 1750 亿。这需要几百张 A100 GPU 跑几个月。

## 小模型的价值

MiniMind 这类项目的意义不在于效果有多强，而在于它把完整的训练流程压缩到了一个你能在本地运行的规模：

- 一个极小的 transformer 模型
- 一个小语料库
- 完整的 tokenizer、训练脚本、推理脚本

你能真实看到 loss 怎么下降，能改一个参数看效果，能理解每一个组件在做什么。这比读一百篇关于 LLM 的文章更有效。

## 预训练之后

预训练结束，你得到的是一个"文字接龙高手"。它非常擅长预测下一个词，但它不一定擅长回答问题、遵循指令、拒绝有害请求。

这就是为什么预训练之后还需要 SFT（指令微调）和各种对齐技术——预训练给了模型知识，对齐让模型知道怎么用这些知识。

## 小结

语言模型的预训练，核心就是一件事：在海量文本上，通过不断预测下一个词来学习语言、知识和结构。

- **Tokenizer** 把文字变成 token
- **Embedding** 把 token 变成向量
- **Attention** 让每个 token 参考其他 token
- **Loss** 衡量预测误差，驱动参数更新

理解这个链路，比记住模型参数量和性能数字有用得多。
